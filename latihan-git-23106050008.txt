3. Apa yang dimaksud dengan WIP (Work In Progress) di dalam Kanban?
- WIP (Work In Progress) adalah jumlah tugas atau item yang sedang dikerjakan pada saat bersamaan. Ini merupakan indikator penting untuk mengelola alur kerja dan mencegah tim menjadi kelebihan beban. Batasan WIP membantu tim untuk fokus pada tugas yang sedang berlangsung dan meningkatkan efisiensi.

4. Sebutkan dan jelaskan jenis version control system yang digunakan dalam pengembangan perangkat lunak?
-Version Control System (VCS) adalah sistem yang digunakan dalam pengembangan perangkat lunak untuk mengelola perubahan terhadap kode sumber dari waktu ke waktu. Secara umum, VCS terbagi menjadi dua jenis utama, yaitu Local Version Control System ,Centralized Version Control System (CVCS) dan Distributed Version Control System (DVCS): 
1. Local Version Control Systems 
Local Version Control Systemsmerupakan jenis VCS paling sederhana dan memiliki database yang menyimpan semua perubahan pada file. 
2. Centralized Version Control System (CVCS)
CVCS adalah sistem kontrol versi di mana seluruh versi kode dan riwayatnya disimpan pada satu server pusat. Pengembang harus terhubung ke server tersebut untuk mengambil salinan kode (checkout) dan mengunggah perubahan (commit). Sistem ini memudahkan pengelolaan dan kontrol terpusat, namun memiliki kelemahan jika server pusat mengalami gangguanâ€”maka seluruh tim tidak dapat bekerja atau bahkan berisiko kehilangan data. Contoh CVCS yang populer adalah Subversion (SVN) dan CVS.
3. Distributed Version Control System (DVCS)
DVCS adalah sistem kontrol versi di mana setiap pengembang memiliki salinan lengkap dari repositori, termasuk seluruh riwayat perubahan. Proses commit dilakukan secara lokal, dan hanya perlu terhubung ke server saat melakukan sinkronisasi melalui push atau pull. Sistem ini lebih fleksibel karena memungkinkan pengembang bekerja secara offline dan memiliki cadangan otomatis dari repositori. DVCS juga lebih aman dan mendukung kolaborasi tim dengan lebih baik. Contoh paling umum dari DVCS adalah Git, yang saat ini menjadi standar industri dalam pengembangan perangkat lunak.
